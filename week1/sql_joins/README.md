# SQL Practice – JOINS

##  Overview

This section explains how to combine data from multiple tables using different types of JOINS.

---

##  Definition

JOINS are used to retrieve related data from two or more tables based on a common column.

---

##  Purpose

* Combine data from multiple tables
* Retrieve meaningful relationships
* Build complex queries

---

##  Types of JOINS

* INNER JOIN → returns matching records from both tables
* LEFT JOIN → returns all records from left table and matching from right
* RIGHT JOIN → returns all records from right table and matching from left
* FULL JOIN → returns all records from both tables
* SELF JOIN → joins a table with itself

---

##  Key Points

* JOIN is used when data is stored in multiple tables
* Matching is done using common columns
* LEFT JOIN helps find missing data
* SELF JOIN is used for hierarchical relationships

---

##  Use Cases

* Employee and department mapping
* Sales and customer analysis
* Finding missing or unmatched records
* Building reports from multiple sources
